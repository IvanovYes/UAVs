# UAVs

Макет ПО управлению группой БВС.
Архитектура ПО показана в следующей структурной схеме:
ПО состоит из двух незаисимых исполняемых частей: 
Сервера - центрального узла, который управляет группой БВС
Клиентов - отдельных экземпляров ПО, запущенных на каждом БВС (на Raspberry Pi или аналогичном бортовом компьютере), и соединяемых с полетным контроллером.
Архитектура работы сервера представлена на следюущей схеме:
Из двух (мо моему мнению) возможных вариантов:
классическа архитектура (сервер слушает, клиент подключается) 
гибридная архитектура (сервер и клиенты взаимно инициируют соединение к друг друга через последовательный вызов команд bind и connect.
Сервер работает в трехпоточном режиме (такая структура была выбрана исходя из того, что : 
главный поток - непрерывно выводит информацию об состоянии группы БВС
принимающий поток - непрерывно принимает данные с клиентов и записывает их в буфер (встроенную (реализованую непосредственно в коде как класс) динамически обновляемую базу данных об задаваемых клиентах-БВС до начало выполнения полетного задания).
управляющий поток - непрерывно обрабатывает информацию из буфера и принимает входящие запросы от пользователя, на основании чего шлет команды всей группе либо отдельному клиенту БВС.
